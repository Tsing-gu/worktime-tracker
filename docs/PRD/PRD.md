# 工时计算器 PRD v2.0

## 1. 产品概述

自动追踪每日工作时长（基于键鼠活动检测），按用户设定的每日工时要求，实时计算本周剩余天数需要达成的工时，并提供下班提醒。

**运行形态**：macOS 本地桌面程序（PySide6），双击即用，零部署。

## 2. 核心概念定义

| 概念 | 定义 |
|------|------|
| 工作日（"一天"） | 早上 6:00 至次日 6:00 的时间窗口（固定北京时间） |
| 键鼠活动 | 通过 `ioreg -c IOHIDSystem` 读取 HIDIdleTime 判断是否有键鼠操作 |
| 上班时间 | 当天 6:00 后 HIDIdleTime 从大于阈值降到小于阈值（首次键鼠活动） |
| 下班时间 | HIDIdleTime 持续超过配置等待时长（用户离开），或手动点击下班 |
| 每日工时 | 下班时间 - 上班时间（毛时间，不扣除中断和午休） |
| 周 | 周一至周日 |
| 每日工时要求 | 用户自定义，默认 8 小时/天 |
| 请假 | 用户标记某天为请假，该天排除出周/月目标天数 |

## 3. 键鼠活动追踪方案

### 3.1 数据采集

通过 `ioreg -c IOHIDSystem` 读取 `HIDIdleTime`（纳秒级空闲时长），每 30 秒轮询一次。

- HIDIdleTime < 5 秒 → 有键鼠活动（刚操作过）
- HIDIdleTime > 5 分钟 → 用户离开

轮询结果写入 SQLite，每条记录包含时间戳和空闲时长。

### 3.2 上班判定

- 时间窗口 6:00 之后，HIDIdleTime 从 >5min 降到 <5s → 记为上班时间
- 上班时间 = 当前轮询时间戳 - HIDIdleTime（回溯到最后一次操作时刻）
- 如果 6:00 时已在操作电脑（HIDIdleTime <5s）→ 上班时间记为 6:00

### 3.3 下班判定（两阶段，阈值可配置）

> 用户可配置：① 离开等待时长（默认 1 小时）② 时间下限（默认 19:00）
> 作用：防止午休/会议等长时间离开被误判为下班

#### 阶段一：实时判定（用于下班提醒）

当时间 ≥ 配置时间下限（默认 19:00），且 HIDIdleTime 持续超过配置等待时长（默认 1 小时）→ 回溯到最后一次键鼠活动时刻，记为**预判下班时间**，触发系统通知。

#### 阶段二：最终判定（次日 6:00 结算）

以工作日时间窗口（6:00-次日6:00）内**最后一次键鼠活动**为准，无论发生在几点。

**边界情况处理**：

| 场景 | 处理方式 |
|------|----------|
| 时间下限（如 19:00）前离开，之后一直未操作 | 按 19:00 记为下班时间 |
| 时间下限后短暂离开（< 配置等待时长）又回来 | 不算下班，继续工作 |
| 通宵工作至次日 2:00 | 仍属于当天工作日（6:00-6:00 窗口），2:00 即为下班时间 |
| 当天完全没有键鼠活动 | 无记录，工时为 0 |
| 一直操作电脑（不离开） | 用户可手动点击"下班"按钮（弹窗确认后记录） |
| 用户手动点击"下班"按钮 | 弹窗确认后，记录当前时间为下班时间，不再追踪当天 |

### 3.4 工时计算

```
每日工时 = 下班时间 - 上班时间（毛时间，不扣除中断和午休）
```

## 4. 工时计算

### 4.1 周工时计算

```
本周总工作日数 = 默认5（周一至周五） - 法定假日数 + 调休上班日数 - 请假天数
本周已工作天数 = 本周已有下班记录的天数（含今天）
本周已工作总时长 = 各已工作日工时之和
本周剩余天数 = 本周总工作日数 - 已工作天数
本周剩余需达标总时长 = 每日工时要求 × 本周总工作日数 - 已工作总时长
本周剩余每天需达成 = 剩余需达标总时长 / 剩余天数
```

### 4.2 月工时计算

```
本月已过工作日 = 当月已过去的实际工作日（去除法定假日+请假，加上调休上班日）
本月已工作总时长 = 各已工作日工时之和
月平均工时 = 本月已工作总时长 / 本月已过工作日数
```

> 不跨周结转，不跨月结转，周和月各自独立计算。

### 5.3 数据存储

- SQLite 本地数据库，文件路径：`~/.worktime_tracker/worktime.db`
- 表结构：

```sql
-- 键鼠活动记录表
CREATE TABLE activity_events (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    timestamp DATETIME NOT NULL,
    idle_seconds REAL NOT NULL,           -- HIDIdleTime（秒）
    is_active INTEGER NOT NULL,           -- 1=有活动(idle<5s), 0=空闲(idle>5min)
    work_date DATE NOT NULL               -- 归属工作日（6:00窗口计算后）
);

-- 每日工时表
CREATE TABLE daily_worktime (
    work_date DATE PRIMARY KEY,
    start_time DATETIME,
    end_time DATETIME,
    total_hours REAL,
    required_hours REAL,
    is_holiday INTEGER DEFAULT 0,
    is_adjusted_workday INTEGER DEFAULT 0,  -- 调休上班日
    leave_type TEXT,                        -- 请假类型：annual/sick/personal/compensatory/none
    is_confirmed INTEGER DEFAULT 0,         -- 用户是否已确认次日提醒
    has_anomaly INTEGER DEFAULT 0,          -- 是否检测到异常
    anomaly_note TEXT,
    source TEXT DEFAULT 'auto',             -- 'auto' | 'manual'（手动补录）
    note TEXT
);

-- 节假日缓存表
CREATE TABLE holidays (
    date DATE PRIMARY KEY,
    name TEXT,
    is_off_day INTEGER NOT NULL             -- 1=放假日, 0=调休上班日
);

-- 设置表
CREATE TABLE settings (
    key TEXT PRIMARY KEY,
    value TEXT
);
```

## 7. 技术方案

### 7.1 技术栈

- **GUI**：PySide6 6.11.1（Qt 6.11.1），原生 macOS 外观 + 菜单栏驻留
- **Python**：conda base Python 3.12.4（`/opt/anaconda3/bin/python`）
- **数据库**：sqlite3（Python 内置）
- **键鼠活动检测**：subprocess 调用 `ioreg -c IOHIDSystem`，读取 HIDIdleTime，每 30 秒轮询
- **菜单栏驻留**：PySide6 QSystemTrayIcon（原生支持，无需额外库）
- **系统通知**：osascript 调用 macOS 通知中心
- **开机自启**：`~/Library/LaunchAgents/com.tsinggu.worktime-tracker.plist`
- **节假日 API**：GitHub `NateScarlet/holiday-cn`，主 URL + jsDelivr CDN 备 + 本地缓存容灾
- **数据导出**：csv 模块（CSV）+ openpyxl（Excel）

### 7.2 后台监控流程

```
程序启动
  → 初始化 SQLite 数据库
  → 读取 ioreg 获取当前 HIDIdleTime
  → 写入 activity_events 表
  → 启动 QTimer 定时轮询（每 30 秒）
    → 读取 HIDIdleTime → 判断活跃/空闲 → 记录事件
    → 检测上班条件（6:00后首次活动）→ 记录上班时间
    → 检测下班条件（配置阈值 + 空闲 > 配置时长）→ 触发系统通知
    → 检测每日工时达标 → 触发弹窗提醒
    → 更新主界面实时数据
  → 跨天结算（检测到日期从6:00跨越）
    → 计算前一天最终上下班时间
    → 检测异常（活动断层过多、上班过早等）
    → 更新 daily_worktime 表
  → 工作日首次打开时
    → 弹窗显示前一天下班时间，用户确认或修改
    → 显示异常提醒（如有）
```

## 8. 已确认事项

### 初始确认

1. **工时计算方式**：毛时间（下班-上班），不扣除中断时长，不扣除午休
2. **下班判定边界**：19:00 前离开后一直未操作 → 按 19:00 记为下班时间
3. **节假日数据源**：API 自动获取中国法定节假日
4. **手动下班**：提供手动"下班"按钮，用于不离开电脑的场景
5. **请假功能**：用户可标记某天为请假（年假/病假/事假等），请假天数不计入剩余工作日数
6. **次日提醒**：每个工作日开始时弹窗提醒用户确认前一天的下班时间，仅工作日提醒（周五的记录周一提醒），可手动调整

### 第1轮

7. **请假与周目标**：请假天排除出本周目标天数（总目标和剩余天数都减1）
8. **手动下班交互**：按下后弹窗确认，确认后记录当前时间为下班时间
9. **次日确认方式**：每个工作日打开电脑时弹窗显示前一天下班时间，用户确认或修改

### 第2轮

10. **上班判定**：6:00 后首次键鼠活动记为上班时间，过滤短暂唤醒
11. **下班判定**：用户可配置阈值（离开等待时长 + 是否设时间下限），默认保留 19:00+1小时方案
12. **节假日 API**：使用免费公网 API 自动获取中国法定节假日
13. **手动补录**：支持手动输入某天的上下班时间，覆盖自动追踪结果

### 第3轮

14. **提醒场景**：① 达到每日工时要求时弹窗提醒 ② 下班时间后系统通知栏提醒
15. **历史记录**：默认日历视图，每天一格显示工时和状态
16. **导出格式**：CSV + Excel 两种格式都支持

### 第4轮

17. **程序驻留**：关闭主窗口后转入菜单栏（右上角图标）继续运行，点击图标重新打开
18. **开机启动**：开机后自动启动并静默运行，无需手动操作
19. **请假录入**：日历右键标记 + 专用请假按钮，两种方式都支持
20. **多周补偿**：不跨周结转，但需同时展示周平均和月平均两个维度的工时数据

### 第5轮

21. **月平均口径**：当月已过工作日的平均工时（未到的工作日不参与计算）
22. **界面展示**：主界面周+月两个卡片并列显示
23. **异常处理**：自动检测异常（上班早于6:00、一天多次活动断层等），次日弹窗提醒时一并提示用户修正
24. **调休识别**：API返回的调休上班日（周末补班）自动算作工作日，法定假日休息日算休息日

### 技术确认
